# คู่มืออัพเดตเว็บ iPES Tournament Leaderboard

คู่มือนี้ครอบคลุม: (1) สิ่งที่เพิ่ม/ตัดออกในรอบอัปเดตนี้ (2) การตั้งค่า Firebase ให้รองรับฟีเจอร์ใหม่ (3) **ขั้นตอนสำคัญที่ต้องทำก่อน deploy** (4) การ deploy ขึ้น GitHub + Vercel (5) วิธีอัปเดต "เนื้อหา" เว็บแบบไม่ต้องแก้โค้ดเลย (6) **ผลตรวจสอบความปลอดภัยของแผนฟรี (Firebase + Vercel)**

## 0. สิ่งที่เพิ่มเข้ามาในรอบนี้ (รอบล่าสุด — ใหญ่สุดเท่าที่เคยอัปเดต)

### รอบนี้: ระบบทีมแยกหน้า + จัดระเบียบเมนู

- **แก้บัค "เพิ่มเติม ▾" ใช้งานไม่ได้จริง**: เมนู "เพิ่มเติม" ที่เพิ่มมาก่อนหน้านี้ กดแล้วเมนูไม่โผล่ขึ้นมาให้เห็น (หรือโผล่แต่ถูกตัดขาดมองไม่เห็น) สาเหตุคือกล่องเมนูถูกวางไว้ข้างในแถบแท็บที่มีการตั้งค่า "เลื่อนแนวนอนได้" (`overflow-x: auto`) ซึ่งทำให้เบราว์เซอร์ตัดส่วนที่ล้นออกมาด้านล่างทิ้งไปด้วย (เป็นกฎ CSS ที่พอตั้งค่าให้เลื่อนแกนหนึ่งได้ อีกแกนจะถูกบังคับให้ตัดขอบไปด้วยเสมอ) แก้โดยย้ายกล่องเมนูออกมาอยู่นอกแถบที่เลื่อนได้ ทำให้ลอยขึ้นมาแสดงผลถูกต้องแล้ว และย้ายปุ่ม "เพิ่มเติม" ออกมาอยู่นิ่งๆข้างแถบเลื่อน ไม่ต้องเลื่อนหาปุ่มนี้อีกต่อไป

- **อันดับทีมแยกหน้า**: แท็บใหม่ "อันดับทีม" จัดอันดับทีม (สำหรับทัวร์นาเมนต์แบบทีม) จากผลแข่งของทีมเอง (ไม่ใช่สถิติสมาชิกรวมแบบสโมสร์) — แชมป์×10 + รองแชมป์×6 + เข้าร่วม×1 + ชนะ×2 เหมือนสูตรคะแนนผู้เล่น แตะทีมเพื่อดูสมาชิกและสถิติยิง/เสียได้
- **จัดระเบียบเมนูใหม่ ลดความรกของแถบแท็บ**: ย้ายแท็บที่ใช้บ่อยน้อยกว่า (สถิติยิง/สตรีค, หอเกียรติยศ, สถิติ, สรุปรายเดือน, กฎการแข่งขัน, ช่วยด้วย) เข้าไปในเมนู "เพิ่มเติม ▾" ที่ปลายแถบแท็บ ส่วนแท็บที่เช็คบ่อย (อันดับ, ห้องรอแมตช์, สโมสร, อันดับทีม, ทัวร์นาเมนต์, ข่าว) ยังอยู่ข้างหน้าตามเดิม ไม่ได้ลบอะไรออก แค่จัดกลุ่มให้ดูไม่รกตา
- **ปุ่ม "🖥️ เปิดโหมดจอทีวี" ย้ายเป็นเฉพาะแอดมินเห็น**: เดิมทุกคนเห็นปุ่มนี้ในแถบแท็บ ทั้งที่เป็นงานตั้งค่าจอแสดงผลของผู้จัดงาน (ไม่ใช่สิ่งที่ผู้เล่นทั่วไปต้องกดเอง) ย้ายไปอยู่ในเมนู "เพิ่มเติม" และให้เห็นเฉพาะตอน login เป็นแอดมินเท่านั้น
- **แก้บัค `colors is not defined`**: ระหว่างพัฒนาเมนู "เพิ่มเติม" เผลอใช้ตัวแปร `colors` ในไฟล์ `app/page.js` ซึ่งไฟล์นี้ไม่ได้ import ตัวแปรนี้ไว้ (ใช้ object สไตล์ของตัวเองชื่อ `s` แทน) — ถ้าไม่แก้ จะทำให้กดเปิดเมนู "เพิ่มเติม" แล้วเว็บพังทันที ตรวจพบและแก้ก่อนส่งให้แล้ว
- **ไม่มี collection ใหม่ในรอบนี้**: ระบบ "อันดับทีม" คำนวณจาก collection `teams`/`matches`/`tournaments` ที่มีอยู่แล้วทั้งหมด แค่คำนวณแบบใหม่ ไม่ต้องแก้ `firestore.rules` เลย

### รอบก่อนหน้า

### ฟีเจอร์ใหม่ล่าสุด

- **รายละเอียดสโมสร์เชิงลึก**: ที่ "Admin → จัดการสโมสร์" เพิ่มฟิลด์ใหม่ — ประวัติ/คำอธิบาย, เมือง/ประเทศ, ลิงก์ Facebook ของสโมสร์ และในหน้า "สโมสร" กดที่สโมสร์ไหนแล้วกด "ดูรายละเอียดสโมสร์ทั้งหมด" จะเห็นหน้าใหม่ — สถิติรวม 4 กล่อง, รายชื่อสมาชิกปัจจุบันทั้งหมด, และ "ผลงานในอดีตของสมาชิก" (ทุกครั้งที่สมาชิกคนใดคนหนึ่งในสโมสร์เคยแชมป์/รองแชมป์ทัวร์นาเมนต์ ไม่ว่าซีซั่นไหน) — ระบบเก็บแค่สังกัดปัจจุบันของผู้เล่น ไม่มีบันทึกประวัติย้ายสโมสร์ ระบุไว้ในหน้าให้ชัดเจน ไม่ใช่ overclaim
- **Export สถิติเจาะลึก CSV**: ที่แท็บ Admin เพิ่มปุ่ม Export 2 ตัวใหม่ — "สถิติเจาะลึก" (ทุกค่าที่คำนวณแล้ว: คะแนน, อัตราชนะ, ยิงได้/เสีย, สตรีค ของทุกคนในตารางเดียว) และ "ประวัติแมตช์ทั้งหมด" (ทุกแมตช์ที่ผู้เล่นแต่ละคนเคยแข่ง พร้อมสถิติเจอกันสะสมกับคู่แข่งคนนั้น) เปิดได้ตรงใน Excel/Numbers/Google Sheets
- **ตัวกรองขั้นสูงใน Global Search**: กดแว่นขยายที่หัวเว็บ จะมีชิปตัวกรองให้เลือก — "เฉพาะสโมสร์เดียวกับคำค้นหา", "ยังไม่เคยแชมป์", "ฟอร์มร้อนแรง 🔥" เลือกตัวกรองได้โดยไม่ต้องพิมพ์คำค้นหาก่อนก็ได้ (ไล่ดูได้เลย)
- **สรุปสถิติประจำเดือน/ฤดูกาล**: แท็บใหม่ "สรุปรายเดือน" แอดมินกด "สร้างสรุปใหม่" ตั้งชื่อ (เช่น "มิถุนายน 2026") ระบบจับภาพ Top 10, คนขยับอันดับมากสุด, ฟอร์มร้อนแรงที่สุด, สโมสร์อันดับ 1 ณ ขณะนั้นเก็บไว้ถาวร ย้อนดูได้ทุกเมื่อ (ไม่ส่งออกไปไหน เก็บในเว็บอย่างเดียว)
- **หน้า "ช่วยด้วย"**: แท็บใหม่สำหรับผู้เล่นมือใหม่ อธิบายกลไกเว็บแบบถาม-ตอบ (อันดับคำนวณยังไง, รหัสยืนยันคู่แข่งคืออะไร, ห้องรอแมตช์ดูตรงไหน ฯลฯ)
- **เมนู TV แยกออกมา**: เพิ่มปุ่ม "🖥️ จอทีวี" ในแถบแท็บ (กรอบเส้นประ) กดแล้วเปิดหน้า `/tv` ในแท็บใหม่
- **แก้ไข UI หลายจุด**: ปรับสีสโมสร์ที่เคยทับชื่อผู้เล่นให้เนียนขึ้น (เป็นวงแหวนสีรอบโลโก้เล็กๆ แทนแถบสีเต็มขอบบน), แก้การ์ดสถิติ 4 กล่องในหน้าสโมสร์ไม่ให้ป้ายชื่อขึ้น 2 บรรทัดเกะกะบนมือถือ, แก้หัวข้อหน้าจอทีวีให้ปรับขนาดตามความกว้างจอแทนใช้ค่าตายตัว (กันพังถ้าเปิดบนมือถือโดยไม่ตั้งใจ), ตรวจทุกจุดที่มีความเสี่ยงข้อความซ้อนทับทั้งระบบ
- **เพิ่มคำอธิบายคะแนนชนะ +2**: แถบคำอธิบายด้านล่างตารางอันดับมีครบทุกองค์ประกอบของสูตรคะแนนแล้ว

### แก้ไขจากรอบก่อนหน้า

- **Modal ซ้อนทับกันที่หน้าจัดการสโมสร์/ทีม** — แก้ให้ปิด-เปิดสลับถูกต้อง
- **Bracket pulse animation ตัดสั้นเมื่อมี 2 แมตช์จบพร้อมกัน** — แก้ให้แต่ละแมตช์มีตัวจับเวลาของตัวเอง
- **`addPlayer`/`updatePlayer` ไม่บันทึก `facebookUrl`** — แก้แล้ว

### ฟีเจอร์รอบก่อนหน้านั้น (เน้นไม่ต้อง Login ฝั่งผู้เล่น)

- **🖥️ จอทีวี (TV Display Mode)**: หน้าแยก `/tv` วนแสดง 4 หน้าอัตโนมัติทุก 12 วินาที
- **ห้องรอแมตช์ (Live Lobby)**: รวมทุกคู่ที่รอแข่งจากทุกทัวร์นาเมนต์ไว้ในที่เดียว
- **ลิงก์/QR ส่วนตัวต่อผู้เล่น**: เปิดโปรไฟล์แล้วกด "คัดลอก"/"QR" แชร์ลิงก์ตรงไปหน้านั้นได้เลย ไม่ต้อง login
- **คะแนนชนะแมตช์ = +2 คะแนน**: เพิ่มเข้าสูตรคะแนนเดิม

ระบบเดิมทั้งหมด (สร้างสาย/จับสาย, กรอกผล, Deadline, สถิติ, Achievement, Compare, Search, Backup & Restore, News, ทีม, สโมสร์) **ยังทำงานปกติทุกอย่าง ไม่กระทบกัน**

### เพิ่ม collection ใหม่ในรอบนี้

ฟีเจอร์สรุปสถิติเพิ่ม collection ใหม่ชื่อ **`summaries`** — ต้อง deploy `firestore.rules` ใหม่ก่อนใช้งาน (ดู §2.1) ไม่ต้องสร้าง collection ล่วงหน้าเอง ระบบสร้างให้อัตโนมัติตอนแอดมินกดสร้างสรุปครั้งแรก ส่วนฟิลด์ใหม่ของสโมสร์ (description/location/facebookUrl) เป็นฟิลด์เพิ่มในเอกสาร `clubs` เดิม ไม่ต้องทำอะไรเพิ่มเลย

### ฟีเจอร์ใหม่

- **iPES Ranking Series**: แก้ชื่อหัวเว็บจาก "iPES" เป็น "iPES Ranking Series"
- **ระบบแข่งแบบทีม (3v3 / 4v4 / 5v5)**: ตอนสร้างทัวร์นาเมนต์ใหม่ เลือกได้ว่า "เดี่ยว" หรือ "ทีม" — ถ้าเลือกทีม แอดมินต้องสร้างทีมไว้ก่อนที่ "Admin → จัดการทีม" (ตั้งชื่อ+โลโก้+เลือกสมาชิกให้ครบขนาดทีม) แล้วเลือกทีมมาจับสายแบบ Bracket แพ้คัดออกเหมือนระบบเดี่ยวทุกอย่าง **ผลแข่งของทีม (แชมป์/รองแชมป์/ชนะ/แพ้/ยิงประตู/ฟอร์ม) จะนับรวมเข้าสถิติส่วนตัวของสมาชิกทุกคนในทีมโดยอัตโนมัติ** ไม่ต้องกรอกซ้ำ
- **ลิงก์ Facebook ผู้เล่น**: แอดมินกรอกลิงก์ Facebook ให้ผู้เล่นแต่ละคนได้ในฟอร์มเพิ่ม/แก้ไขผู้เล่น ขึ้นเป็นไอคอนกดเปิดได้ทันทีในหน้าโปรไฟล์
- **คู่ปรับตลอดกาล (Rivalry Tracker)**: แท็บย่อยใหม่ในหน้า "สถิติยิง/สตรีค" แสดงคู่ที่เจอกันบ่อยและสถิติสูสีที่สุด พร้อมประวัติการเจอกันแบบเจาะลึก
- **ตารางฟอร์ม 5 นัดล่าสุด (Form Table)**: แท็บย่อยใหม่ จัดอันดับจากฟอร์ม 5 นัดหลังสุดแบบลีกฟุตบอลจริง (ชนะ=3 แต้ม) แยกจากอันดับคะแนนสะสมหลัก
- **สีธีมประจำสโมสร์ (Custom Team Banner)**: ตั้งสีหลัก/สีรองให้สโมสร์ได้ที่ "Admin → จัดการสโมสร์" สีจะไปขึ้นเป็นแถบ/แบนเนอร์บนการ์ดผู้เล่น Podium, หน้าอันดับสโมสร์, และหัวโปรไฟล์ผู้เล่นที่สังกัดสโมสร์นั้น
- **Stadium Atmosphere**: หัวโปรไฟล์ผู้เล่นมีออร่าเปลี่ยนตามฟอร์มปัจจุบันแบบอัตโนมัติ — ชนะติดต่อกัน 5+ = ออร่าสีแดงร้อนแรง 🔥, ชนะ 3-4 = ออร่าทอง ✨, แพ้ 3 จาก 3 นัดหลังสุด = ออร่าฟ้าเย็นชา 🧊 ไม่ต้องตั้งค่าอะไร คำนวณจากสถิติที่มีอยู่แล้ว
- **Bracket แบบมีแอนิเมชัน**: เส้นเชื่อมระหว่างรอบในตารางสายวาดด้วย SVG พร้อมแอนิเมชันลากเส้น และการ์ดแมตช์ที่เพิ่งกรอกผลจะมีจังหวะเรืองแสงสั้นๆ (pulse) ให้เห็นว่า "เพิ่งจบไป" — เป็น CSS/SVG ล้วน ไม่ใช้ 3D library หนักๆ เพื่อให้ยังโหลดเร็วบนมือถือและไม่กระทบแผนฟรี
- **ดาวน์โหลดตารางสายการแข่งขัน (PNG/PDF)**: ในหน้า "ทัวร์นาเมนต์" เปิดดูตารางสายแข่งไหนอยู่ จะมีปุ่ม "ดาวน์โหลดรูป (PNG)" และ "พิมพ์ / บันทึก PDF" ให้แอดมินเท่านั้น — PNG ได้ไฟล์รูปพร้อมแชร์ทันที ส่วน PDF เปิดหน้าต่างสั่งพิมพ์ของเบราว์เซอร์ ซึ่งบน iPad Safari จะมีตัวเลือก "บันทึกเป็น PDF" ในหน้าแชร์ ไม่ต้องเพิ่ม dependency อะไรเลย
- **โลโก้สโมสร์ในหน้าอันดับ**: แอดมินจัดการโลโก้สโมสร์ได้ที่ "Admin → จัดการสโมสร์" จับคู่กับชื่อสโมสร์ที่กรอกให้ผู้เล่นอัตโนมัติ ขึ้นแสดงทั้ง Podium, ตารางอันดับหลัก, และหน้าสโมสร์
- **เปลี่ยนโลโก้เว็บ**: เปลี่ยนจากไอคอน SVG เดิมเป็นรูปภาพโลโก้ที่อัปโหลดเอง (ไฟล์ `public/logo.png`)
- **Recent Matches + Upcoming Tournament**: วิดเจ็ตแมตช์ล่าสุด/ทัวร์นาเมนต์ที่กำลังจะมาถึง แสดงบนแท็บ "อันดับ"
- **Goals Ranking + Win Streak Ranking**: แท็บย่อยอันดับยิงประตูสูงสุด และสตรีคชนะติดต่อกันสูงสุด
- **Hall of Fame แยกตามซีซั่น**: สลับดู "แชมป์ตลอดกาล" หรือ "แยกตามซีซั่น" ได้
- **กฎการแข่งขัน (Rule)**: แท็บใหม่ แอดมินพิมพ์/แก้ไขกฎได้ตรงในเว็บเลย
- **Match Verification Code**: ทุกแมตช์ในสายแข่งได้รหัสยืนยันคู่แข่งขันอัตโนมัติ (`IPES-XXXX`) กดคัดลอกได้

ระบบเดิมทั้งหมด (สร้างสาย/จับสาย, กรอกผล, Deadline, สถิติ, Achievement, Compare, Search, Backup & Restore, News) **ยังทำงานปกติทุกอย่าง ไม่กระทบกัน**

### สิ่งที่ตรวจสอบแล้วและ "ไม่ต้องลบ" (ดูเหตุผลเต็มที่ §6)

รอบนี้ตรวจทั้งระบบตามที่ขอ — **ไม่พบจุดที่ต้องลบหรือต้องแก้เพื่อความปลอดภัยของแผนฟรี** ทุกฟีเจอร์ใหม่และของเดิมยังคงสอดคล้องกับหลักการเดิม: คำนวณสดที่เบราว์เซอร์, ไม่มี polling/interval, ไม่มี Cloud Function, ไม่เก็บรูปภาพเป็นไฟล์ใน Firestore/Storage (ใช้ลิงก์ภาพนอกระบบทั้งหมด) ดูรายละเอียดที่ §6

### เพิ่ม collection ใหม่ใน Firestore

ฟีเจอร์รอบนี้เพิ่ม collection ใหม่ 2 ตัว: **`clubs`** (โลโก้/สีธีมสโมสร์) และ **`teams`** (ทีมสำหรับทัวร์นาเมนต์แบบทีม) — ต้อง deploy `firestore.rules` ใหม่ก่อนใช้งาน (ดู §2.1) ไม่ต้องสร้าง collection ล่วงหน้าเอง ระบบสร้างให้อัตโนมัติตอนแอดมินกดเพิ่มสโมสร์/ทีมครั้งแรก

ฟีเจอร์ดาวน์โหลดตารางสาย (PNG/PDF) และฟีเจอร์ Animated Bracket **ไม่ต้องเพิ่ม npm package ใหม่เลย** — ใช้ความสามารถที่มีในตัวเบราว์เซอร์ (Canvas API, SVG, CSS Animation, Print dialog) ล้วนๆ แค่ `git pull` โค้ดใหม่แล้ว build/deploy ตามปกติก็ใช้ได้เลย

### หน้า `/tv` (จอทีวี) — ไม่ต้องตั้งค่าอะไรเพิ่ม

หน้านี้เป็นไฟล์ใหม่ `app/tv/page.js` deploy ขึ้น Vercel พร้อมกับเว็บหลักโดยอัตโนมัติ เข้าได้ทันทีที่ `https://โดเมนเว็บคุณ/tv` (เช่น `https://i-pes-website.vercel.app/tv`) ไม่ต้องสร้าง Firestore collection ใหม่ ไม่ต้องตั้งค่า Environment Variable เพิ่ม — ใช้ข้อมูลเดียวกับเว็บหลักทั้งหมด

### QR Code ลิงก์ส่วนตัวผู้เล่น — ไม่ต้องเพิ่ม npm package

ใช้ public API ของ `api.qrserver.com` (เว็บฟรี ไม่ต้องสมัครบัญชี ไม่ต้องขอ API key) สร้างรูป QR แบบ on-the-fly จากลิงก์โปรไฟล์ ไม่ต้องเพิ่ม dependency ใดๆ ในโค้ดเลย

### รอบก่อนหน้า (สำหรับอ้างอิง)

ระบบจัดทัวร์นาเมนต์แบบสายแข่งจริง (Bracket / Single Elimination) ที่ทำงานครบวงจร:

- **สร้างสาย + จับสายแบบสุ่ม**: เลือกผู้เล่นจากฐานข้อมูล + ขนาดสาย (8/16/32/64/128) ระบบจับคู่สุ่มให้อัตโนมัติ ผู้เล่นไม่ครบจะได้ BYE ผ่านเข้ารอบถัดไปอัตโนมัติ
- **กรอกผล + เลื่อนสายอัตโนมัติ**: แอดมินกรอกสกอร์ (มีจุดโทษถ้าเสมอ) ผู้ชนะจะถูกเลื่อนเข้ารอบถัดไปให้อัตโนมัติ ไม่ต้องกดอะไรเพิ่ม จนถึงประกาศแชมป์/รองแชมป์ของทัวร์นาเมนต์เมื่อจบรอบชิง
- **Deadline ต่อรอบ + แก้ผลย้อนหลัง**: ตั้งเวลาปิดรับผลแต่ละรอบได้ ถ้าเลยเวลาแอดมินเลือก "ผู้ชนะ" ตรงได้โดยไม่ต้องมีสกอร์ และแก้ผลแมตช์ที่จบไปแล้วได้ — ระบบจะล้างผลทุกรอบถัดไปที่อิงจากผลเดิมให้อัตโนมัติ (รวมถึงถอนสถานะแชมป์ ถ้าแก้ไปถึงรอบชิง)
- **สถิติคำนวณสด**: ชนะ/แพ้/ประตู/Win Streak/Head-to-Head/ฟอร์มล่าสุด/**เทรนด์อัตราชนะ** (กราฟเส้นเล็ก 7/30/90 วัน/ทั้งหมด) คำนวณจากแมตช์จริงทุกครั้งที่โหลดหน้า ไม่มีการบันทึกตัวเลขซ้ำซ้อนไว้ที่ไหน
- **Achievement/Badge**: ติดป้ายอัตโนมัติ (แชมป์ครั้งแรก, ชนะติดต่อกัน 5/10, Giant Killer, แชมป์ติดต่อกัน ฯลฯ) ในหน้าโปรไฟล์ผู้เล่น
- **ไฮไลท์ Rivalry/Upset**: หน้าสถิติจะโชว์คู่ปรับที่เจอกันบ่อยและสูสี และแมตช์ที่ผู้เล่นอันดับต่ำกว่าพลิกชนะผู้เล่นอันดับสูงกว่า
- **เปรียบเทียบผู้เล่น (Compare)**: เทียบสถิติ 2 คนแบบเคียงข้าง + ประวัติการเจอกัน กดได้จากปุ่มในหน้าโปรไฟล์
- **ค้นหาแบบรวม (Global Search)**: ไอคอนแว่นขยายที่หัวเว็บ ค้นหาผู้เล่น/ทัวร์นาเมนต์/ข่าวพร้อมกันได้ในที่เดียว
- **Backup & Restore**: หน้า Admin มีปุ่ม Export ข้อมูลทั้งหมดเป็นไฟล์ JSON (และ Export CSV เฉพาะผู้เล่น) เก็บไว้ได้ทุกเมื่อ และนำไฟล์ JSON กลับมา Import (เขียนทับเฉพาะรายการที่ ID ตรงกัน ไม่ลบของเดิม) ใช้กรณีข้อมูลผิดพลาดและต้องการย้อนคืน
- **แดงแจ้งข่าวใหม่**: จุดสีแดงเล็ก ๆ ข้างแท็บ "ข่าว" เมื่อมีประกาศใหม่ที่ยังไม่ได้เปิดดู

ทัวร์นาเมนต์แบบเดิม (กรอกแชมป์/รองแชมป์ด้วยมือ สำหรับผลย้อนหลังก่อนมีระบบนี้) **ยังใช้งานได้ตามปกติ 100%** — ตอนกดปุ่ม "เพิ่มทัวร์นาเมนต์" จะมีให้เลือก 2 แบบคือ "สร้างสายแข่งใหม่" (ระบบใหม่) กับ "บันทึกผลย้อนหลัง" (แบบเดิม)

ถ้า build ผ่านบนเครื่อง ก็ push ได้อย่างมั่นใจ

## 2. ตั้งค่า Firebase

### 2.1 Deploy กฎ Firestore ใหม่ (ถ้ายังไม่ได้ deploy จากรอบก่อนหน้า)

ไฟล์ `firestore.rules` มี collection **`matches`** (เก็บผลแต่ละแมตช์ในสายแข่ง รวมถึงรหัสยืนยันคู่แข่งขัน `verifyCode` ที่เป็นแค่ field หนึ่งในเอกสารเดิม ไม่ต้องเพิ่มกฎอะไรใหม่) และ doc **`meta/rule`** (เนื้อหากฎการแข่งขัน) โดยใช้กฎเดียวกับของเดิม (อ่านได้ทุกคน, เขียนได้แค่ admin) ถ้ายังไม่เคย deploy กฎไฟล์นี้มาก่อนเลย ให้ทำตามนี้:

```
firebase deploy --only firestore:rules
```

(ต้องมี Firebase CLI ติดตั้งและ login ไว้แล้ว ถ้ายังไม่มีให้ `npm install -g firebase-tools` แล้ว `firebase login` และ `firebase use <project-id>` ก่อน)

หรือจะ copy เนื้อหาไฟล์ `firestore.rules` ไปแปะในหน้า Firestore > Rules บน Firebase Console แล้วกด Publish ก็ได้เช่นกัน ไม่ต้องใช้ CLI

### 2.2 ไม่ต้องสร้าง collection/document ล่วงหน้า

`matches`, `clubs`, `teams`, `summaries`, `meta/rankingSnapshot`, และ `meta/rule` ทั้งหมดจะถูกสร้างขึ้นอัตโนมัติทันทีที่แอดมินใช้งานฟีเจอร์นั้นครั้งแรก (จับสายครั้งแรก / เพิ่มสโมสร์ครั้งแรก / สร้างทีมครั้งแรก / สร้างสรุปสถิติครั้งแรก / กดบันทึกอันดับครั้งแรก / บันทึกกฎการแข่งขันครั้งแรก) ไม่ต้องสร้างเองใน Console

### 2.3 ผู้เล่น/ทัวร์นาเมนต์เดิมที่มีอยู่แล้ว

ข้อมูลเดิมทั้งหมด (ผู้เล่น, ทัวร์นาเมนต์แบบกรอกมือ, ข่าว) ใช้งานได้ตามปกติ 100% ไม่ต้อง migrate อะไร ระบบจะรวมสถิติจากทัวร์นาเมนต์แบบสายแข่งใหม่เข้ากับตัวเลขเดิมให้อัตโนมัติทุกครั้งที่โหลดหน้า (คะแนน/แชมป์/รองแชมป์/Badge/อันดับยิงประตู/อันดับสตรีค จะถูกต้องโดยไม่ต้องแก้อะไรเพิ่ม) ส่วนรหัสยืนยันคู่แข่งขัน (`verifyCode`) จะมีเฉพาะแมตช์ที่จับสายตั้งแต่รอบอัปเดตนี้เป็นต้นไป — สายแข่งเก่าที่จับไว้ก่อนหน้านี้จะไม่มีรหัสนี้ (การ์ดแมตช์จะไม่โชว์ช่องรหัสถ้าไม่มี ไม่กระทบการใช้งานอื่น) ผู้เล่นเดิมที่กรอกชื่อสโมสร์ไว้แล้วจะขึ้นโลโก้ให้อัตโนมัติทันทีที่แอดมินสร้างสโมสร์ที่มีชื่อตรงกัน ไม่ต้องไปแก้ข้อมูลผู้เล่นทีละคน

## 4. Deploy ขึ้น Vercel

ถ้าโปรเจกต์เชื่อมกับ Vercel อยู่แล้ว (auto-deploy จาก GitHub) แค่ push ตามขั้นตอนที่ 3 ก็พอ Vercel จะ build และ deploy ให้อัตโนมัติ

ถ้ายังไม่ได้เชื่อม:

1. เข้า [vercel.com](https://vercel.com) → New Project → เลือก repo นี้จาก GitHub
2. ตั้งค่า Environment Variables ให้ตรงกับ `.env.local` ของเครื่อง (คีย์ทั้งหมดที่ขึ้นต้นด้วย `NEXT_PUBLIC_FIREBASE_`) — ต้องกรอกทีละตัวในหน้า Settings → Environment Variables ของโปรเจกต์ Vercel
3. กด Deploy

หลัง deploy ครั้งแรกแล้ว ทุกครั้งที่ push โค้ดใหม่ขึ้น branch `main`, Vercel จะ build และอัปเดตเว็บที่ live URL ให้อัตโนมัติ

## 5. สร้างบัญชี Admin

## 7. อัปเดตเนื้อหาเว็บ — **ไม่ต้อง push/deploy ใหม่**

ข้อมูลทั้งหมดที่เปลี่ยนบ่อย ๆ จัดการได้จากหน้าเว็บโดยตรง ผ่านปุ่ม login Admin มุมขวาบน **ไม่ต้องแก้โค้ดหรือ deploy ใหม่เลย**

⚠️ **เมนูจัดใหม่รอบนี้**: แท็บที่ใช้บ่อย (อันดับ, ห้องรอแมตช์, สโมสร, อันดับทีม, ทัวร์นาเมนต์, ข่าว) อยู่แถวหน้าเหมือนเดิม ส่วนแท็บอื่น (สถิติยิง/สตรีค, หอเกียรติยศ, สถิติ, สรุปรายเดือน, กฎการแข่งขัน, ช่วยด้วย, **Admin**) ย้ายไปอยู่ใต้ปุ่ม **"เพิ่มเติม ▾"** ที่ปลายแถบแท็บ — ถ้าหาแท็บที่เคยใช้ไม่เจอ ให้กดปุ่มนี้ก่อน ส่วนปุ่ม "🖥️ เปิดโหมดจอทีวี" ก็ย้ายไปอยู่ในเมนูนี้ด้วย และ**เห็นเฉพาะตอน login เป็นแอดมิน**เท่านั้น

อัปเดตผ่าน Admin UI ได้ทันที (real-time):
- เพิ่ม/แก้ไข/ลบผู้เล่น, รูปการ์ดผู้เล่น, ลิงก์ Facebook — ทำที่แท็บ "อันดับ"
- จัดการโลโก้+สีธีม+ประวัติ+เมือง/ประเทศ+ลิงก์ Facebook ของสโมสร์ — ทำที่ "เพิ่มเติม" → "Admin" → "จัดการสโมสร์"
- สร้างทีม/แก้ไขสมาชิกทีม (สำหรับทัวร์นาเมนต์แบบทีม) — ทำที่ "เพิ่มเติม" → "Admin" → "จัดการทีม" ส่วนอันดับทีมที่คำนวณแล้วดูได้ที่แท็บ "อันดับทีม" (อยู่แถวหน้า)
- สร้างสายแข่งใหม่ทั้งแบบเดี่ยวและแบบทีม + จับสาย (ได้รหัสยืนยันคู่แข่งขันให้อัตโนมัติทุกคู่), กรอกผล/แก้ผลย้อนหลัง, ตั้ง Deadline ต่อรอบ — ทำที่แท็บ "ทัวร์นาเมนต์" → เพิ่มทัวร์นาเมนต์ → สร้างสายแข่งใหม่ → เลือก "เดี่ยว"/"ทีม"
- เพิ่ม/แก้ไข/ลบทัวร์นาเมนต์แบบกรอกมือ (ผลย้อนหลัง) — ทำที่แท็บ "ทัวร์นาเมนต์" เช่นกัน
- ประกาศ/แก้ไข/ลบข่าว — ทำที่แท็บ "ข่าว" (ระบบจะประกาศข่าวอัตโนมัติให้เองเมื่อมีผลรอบรองชนะเลิศ/ชิงชนะเลิศของสายแข่ง)
- เขียน/แก้ไขกฎการแข่งขัน — ทำที่ "เพิ่มเติม" → "กฎการแข่งขัน" พิมพ์/แก้เนื้อหาได้ตรงในเว็บเลย ไม่ต้องแตะโค้ด
- สร้าง/ลบสรุปสถิติประจำเดือน — ทำที่ "เพิ่มเติม" → "สรุปรายเดือน" กด "สร้างสรุปใหม่" ตั้งชื่อแล้วบันทึกได้เลย
- Export CSV เจาะลึก (สถิติทุกคน/ประวัติแมตช์ทั้งหมด) — ทำที่ "เพิ่มเติม" → "Admin" ส่วน Backup & Restore
- บันทึก Ranking Snapshot (เพื่อให้ลูกศร ▲▼ ความเคลื่อนไหวอันดับทำงาน) และ Backup/Restore ข้อมูล (รวมสโมสร์+ทีม+กฎการแข่งขัน+สรุปสถิติด้วย) — ทำที่ "เพิ่มเติม" → "Admin"

แท็บ "สถิติยิง/สตรีค" (ยิงประตู, สตรีคชนะ, ฟอร์ม 5 นัดล่าสุด, คู่ปรับตลอดกาล), "อันดับทีม", และ "หอเกียรติยศ" (ทั้งแบบตลอดกาลและแยกซีซั่น) อัปเดตให้อัตโนมัติทุกครั้งที่มีผลแมตช์ใหม่หรือทัวร์นาเมนต์จบ ไม่ต้องกดอะไรเพิ่ม สถิติของผู้เล่นที่แข่งในทีมก็รวมเข้าตัวเลขส่วนตัวให้อัตโนมัติเช่นกัน แท็บ "ช่วยด้วย" เป็นเนื้อหาคงที่เขียนไว้ในโค้ด — ถ้าอยากแก้ข้อความ FAQ ต้องแก้ไฟล์ `components/Help.js` แล้ว push ใหม่

ต้อง push code + deploy ใหม่ เฉพาะกรณี: แก้ดีไซน์/สี/เลย์เอาต์, เปลี่ยนสูตรคำนวณคะแนน, เพิ่มฟีเจอร์ใหม่ หรือแก้บั๊กในโค้ด — กรณีเหล่านี้เท่านั้นที่ต้องทำตามขั้นตอนที่ 3-4 ซ้ำ

ทุกอย่างในตาราง "สิ่งที่ตัด/ปรับให้เรียบง่ายลง" ที่ §0 สามารถทำเป็นเวอร์ชันเต็มได้ในอนาคตหากเว็บเติบโตจนคุ้มที่จะอัปเกรดเป็น Firebase Blaze plan (จ่ายตามใช้งาน) เช่น Cloud Functions สำหรับคำนวณ/แจ้งเตือนแบบ push จริง หรือเก็บ snapshot อันดับอัตโนมัติทุกวันเพื่อทำกราฟ Ranking History แบบเต็มรูปแบบ — ไม่กระทบกับการใช้งานปัจจุบันถ้ายังไม่ทำ

## 9. ทำทุกขั้นตอนนี้จาก iPad ได้ไหม (ไม่มีคอมพิวเตอร์/Terminal)

ได้ครับ ขั้นตอนที่ §1–§2 ปกติต้องใช้ Terminal (build ทดสอบ, git push, `firebase` CLI) ทำผ่าน Safari บน iPad ได้ทั้งหมด ด้วย 2 วิธีนี้:

### วิธีแนะนำ: GitHub Codespaces (เปิดผ่าน Safari ฟรี ไม่ต้องลงแอป)

ให้คอมพิวเตอร์เปล่า ๆ บนคลาวด์พร้อม VS Code + Terminal เต็มรูปแบบ เปิดในแท็บ Safari — ทำให้ build ทดสอบและ push ได้จริงเหมือนใช้คอมพิวเตอร์

1. แตะไฟล์ zip ที่ได้รับในแอป **Files** แล้วแตะค้างเลือก "แตกไฟล์ Zip" (Uncompress)
2. เข้า **github.com** ด้วย Safari → ล็อกอิน → เปิด repo เดิมที่ใช้กับเว็บนี้ (หรือสร้าง repo ใหม่ถ้ายังไม่มี)
3. ที่หน้า repo กดปุ่มสีเขียว **Code** → แท็บ **Codespaces** → **Create codespace on main** รอประมาณ 1-2 นาที จะได้หน้า VS Code เต็มรูปแบบพร้อม Terminal ด้านล่าง
4. เปิด Files app แบบ Split View ข้าง Safari แล้วลากไฟล์/โฟลเดอร์จาก `ipes` ที่แตกไว้ไปวางทับใน File Explorer ฝั่งซ้ายของ Codespace (ลากทับโฟลเดอร์ `app`, `components`, `lib` และไฟล์ที่แก้ เช่น `firestore.rules`, `DEPLOY_GUIDE.md`, `.gitignore` — **ข้ามไฟล์ `.env.local`**, ไม่ต้องอัปขึ้น GitHub เพราะมีคีย์ลับ — โฟลเดอร์นี้มี `.gitignore` ป้องกัน `.env.local` ไม่ให้ถูก commit โดยไม่ตั้งใจอยู่แล้ว แต่ปลอดภัยกว่าถ้าไม่ลากไฟล์นั้นไปเลยตั้งแต่ต้น)
5. พิมพ์ใน Terminal ของ Codespace:
   ```
   npm install
   npm run build
   ```
   ถ้า build ผ่าน (ไม่มี error สีแดง) แปลว่าโค้ดพร้อม deploy
6. commit + push จาก Terminal เดียวกัน:
   ```
   git add .
   git commit -m "Add team mode, club banners, stadium atmosphere, animated bracket, rivalry tracker, form table, facebook links"
   git push
   ```
7. (ถ้าอยาก deploy firestore rules ผ่าน terminal นี้ด้วยเลย):
   ```
   npm install -g firebase-tools
   firebase login --no-localhost
   firebase deploy --only firestore:rules
   ```
   คำสั่ง `firebase login --no-localhost` จะให้ลิงก์มาเปิดในแท็บ Safari อีกแท็บเพื่อล็อกอิน แล้ว copy โค้ดยืนยันกลับมาวางที่ Terminal

### วิธีลัด: ไม่ต้องใช้ Codespaces เลย (เร็วกว่า แต่ไม่ได้ build ทดสอบก่อน)

- **Firestore rules**: เปิดไฟล์ `firestore.rules` ในแอป Files (แตะดูตัวอย่าง หรือเปิดด้วยแอปข้อความ) → คัดลอกทั้งหมด → เข้า Firebase Console บน Safari → Firestore Database → แท็บ **Rules** → ลบของเดิม วางอันใหม่ → **Publish** (ไม่ต้องใช้ CLI เลย วิธีนี้ใช้แทนขั้นตอนที่ §2.1 ได้)
- **GitHub โดยไม่ผ่าน Codespaces**: ที่หน้า repo บน Safari กด "Add file" → "Upload files" → ลากไฟล์/โฟลเดอร์จากแอป Files มาวาง → Commit (Safari บน iPad รองรับการลากทั้งโฟลเดอร์ได้ แต่ถ้าโครงสร้างไม่ตรง ให้อัปทีละโฟลเดอร์ย่อยแทน) — **ข้าม `.env.local` เช่นเดิม**
- **Vercel**: ถ้า repo เชื่อมกับ Vercel อยู่แล้ว แค่ push โค้ดขึ้น GitHub ด้วยวิธีใดวิธีหนึ่งข้างต้น Vercel จะ build และ deploy ให้อัตโนมัติ ดูผลได้ที่ vercel.com ในแท็บ Safari — ถ้า build fail จะเห็น log สีแดงบอกบรรทัดที่ผิด ส่ง log นั้นมาให้ผมดูต่อได้
- **Environment Variables**: ห้ามอัปไฟล์ `.env.local` ขึ้น GitHub เด็ดขาด ให้เปิดไฟล์นี้ดูค่าผ่านแอป Files แล้วพิมพ์ค่าทีละตัวเข้า Vercel → Settings → Environment Variables เอง (ทำครั้งเดียวตอนตั้งโปรเจกต์ ถ้าทำไปแล้วไม่ต้องทำซ้ำ)

แนะนำให้ใช้ Codespaces (วิธีแรก) ถ้าเป็นไปได้ เพราะได้ build ทดสอบจริงก่อน push ลดโอกาส deploy แล้วพังที่ Vercel

## 10. ผลตรวจสอบความปลอดภัยของแผนฟรี (Firebase Spark + Vercel Hobby)

ตรวจทั้งระบบ (เก่า+ใหม่) ตามที่ขอ — **สรุปผล: ไม่พบจุดที่ต้องลบหรือแก้ ทุกอย่างยังปลอดภัยกับแผนฟรีในระยะยาว** รายละเอียดที่ตรวจ:

| ตรวจหัวข้อ | ผลตรวจ |
|---|---|
| Polling / setInterval ที่ดึงข้อมูลซ้ำๆ | **ไม่มี** — ใช้ `onSnapshot` (real-time listener) ทั้งหมด 10 ตัว (เพิ่ม `summaries`) แต่ละตัวตั้งครั้งเดียวตอนเปิดเว็บ คิด read ตามจำนวนเอกสารที่เปลี่ยนแปลงจริงเท่านั้น ไม่ใช่ polling (ส่วนหน้า `/tv` มี `setInterval` อยู่ 1 ตัว แต่เปลี่ยนแค่ตัวแปรในเบราว์เซอร์ ไม่ได้ดึงข้อมูลใหม่จาก Firestore ไม่กิน quota เพิ่ม) |
| การเขียนข้อมูล (writes) ที่เกิดขึ้นอัตโนมัติโดยไม่มีคนกด | **ไม่มี** — ทุก `addDoc`/`updateDoc`/`deleteDoc`/`setDoc` อยู่ในฟังก์ชันที่ผูกกับปุ่มกดของแอดมินเท่านั้น ไม่มีจุดไหนเขียนข้อมูลอัตโนมัติตามเวลา รวมถึงปุ่ม "สร้างสรุปใหม่" ที่เพิ่มมารอบนี้ก็เขียนแค่ตอนแอดมินกดเองเท่านั้น |
| Cloud Functions / Cloud Scheduler | **ไม่ใช้** — สถิติ/เทรนด์/Deadline ทั้งหมดคำนวณสดที่เบราว์เซอร์ทุกครั้งที่โหลดหน้า ไม่ต้องอัปเกรดเป็น Firebase Blaze plan |
| รูปภาพ (โลโก้สโมสร์/ทีม/avatar) เก็บที่ไหน | **เก็บแค่ลิงก์ (URL) ไม่เก็บไฟล์รูปจริงใน Firestore/Storage เลย** — ทุกรูปเป็นลิงก์ไปเว็บฝากรูปภายนอก (เช่น Imgur) ทำให้ไม่กิน Cloud Storage quota และไม่กิน Firestore document size limit (1 MB/doc) แม้จะมีรูปจำนวนมาก |
| ระบบทีมใหม่ (Team Mode) ทำให้ข้อมูลโตเร็วขึ้นไหม | **ไม่** — การ "ขยาย" ผลแข่งทีมไปเป็นสถิติส่วนตัวของสมาชิกทำในหน่วยความจำฝั่งเบราว์เซอร์ชั่วคราวเท่านั้น (ไม่เขียนกลับ Firestore) จำนวนเอกสารจริงใน `matches` ยังเท่าเดิม (1 เอกสารต่อ 1 คู่แข่งในสาย ไม่ว่าจะเป็นทีมกี่คนก็ตาม) |
| แอนิเมชันใหม่ (Animated Bracket, Stadium Atmosphere) กิน CPU/แบตมือถือเยอะไหม | **น้อยมาก** — ทั้งหมดเป็น CSS `@keyframes` ที่เบราว์เซอร์จัดการเองแบบมีประสิทธิภาพ ไม่มี `requestAnimationFrame` ที่รันต่อเนื่องไม่หยุดเลยในทั้งโปรเจกต์ |
| Export CSV เจาะลึก (สถิติ/ประวัติแมตช์) กินทรัพยากรเซิร์ฟเวอร์ไหม | **ไม่กินเลย** — ทำงานทั้งหมดในเบราว์เซอร์ของแอดมินเอง (เบราว์เซอร์ประมวลผล + สร้างไฟล์ ไม่ส่งอะไรกลับ Firestore เพิ่ม) ต่อให้มีผู้เล่น/แมตช์หลักพันรายการก็ยังเร็ว เพราะข้อมูลถูกโหลดมาอยู่ในเบราว์เซอร์อยู่แล้วตั้งแต่เปิดหน้าเว็บ |
| สรุปสถิติประจำเดือน (`summaries`) จะทำให้ฐานข้อมูลโตเร็วผิดปกติไหม | **ไม่** — 1 สรุป = 1 เอกสารเล็กๆ (ไม่ถึง 5 KB) ต่อให้สร้างทุกเดือนเป็นเวลา 10 ปี ก็ยังไม่ถึง 1 MB รวมทั้งหมด |
| Firestore document count จะโตจนเกินฟรีไหม | ผู้เล่น/สโมสร์/ทีม กิน <1KB ต่อเอกสาร ต่อให้มีผู้เล่น 10,000+ คน สโมสร์/ทีมหลักร้อย ก็ยังไม่ถึง 1% ของ 1GB storage ฟรี ส่วน `matches` โตตามจำนวนแมตช์ที่แข่งจริง (ไม่ใช่ตามเวลา) ซึ่งโตช้ากว่า quota การอ่าน/เขียนรายวันมาก |

**สรุป**: ด้วยสเกล "กลุ่ม/ชุมชน" ที่เว็บนี้ออกแบบมารองรับ (ดูตัวเลขประมาณการที่เคยให้ไว้ — รับผู้เล่นหลักพันคนเปิดเว็บทุกวันได้สบายๆ) ฟีเจอร์ทั้งหมดในรอบนี้ไม่ได้เพิ่มความเสี่ยงให้เว็บล่มหรือโดนตัดการเข้าถึงเพราะใช้งานเกิน quota เลย ยังเปิดฟรีได้ยาวๆ ตามเดิม **ไม่มีระบบไหนที่ "หนักเกินไปสำหรับแผนฟรี" ที่ควรเอาออก**
