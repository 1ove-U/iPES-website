# คู่มืออัพเดตเว็บ iPES Tournament Leaderboard

คู่มือนี้ครอบคลุม: (1) สิ่งที่เพิ่ม/ตัดออกในรอบอัปเดตนี้ (2) การตั้งค่า Firebase ให้รองรับฟีเจอร์ใหม่ (3) **ขั้นตอนสำคัญที่ต้องทำก่อน deploy** (4) การ deploy ขึ้น GitHub + Vercel (5) วิธีอัปเดต "เนื้อหา" เว็บแบบไม่ต้องแก้โค้ดเลย

---

## 0. สิ่งที่เพิ่มเข้ามาในรอบนี้ (รอบล่าสุด)

- **Recent Matches**: วิดเจ็ตแมตช์ล่าสุดที่จบไปแล้ว แสดงที่ด้านบนของแท็บ "อันดับ" ทุกครั้งที่มีผลแมตช์ใหม่จากระบบสายแข่ง
- **Upcoming Tournament**: วิดเจ็ตทัวร์นาเมนต์ที่กำลังแข่งอยู่/ยังไม่ประกาศผล แสดงคู่กับ Recent Matches ในแท็บ "อันดับ" กดแล้วเด้งไปแท็บทัวร์นาเมนต์ได้เลย
- **Goals Ranking + Win Streak Ranking**: แท็บใหม่ "สถิติยิง/สตรีค" มีสับแท็บย่อย 2 อัน — อันดับยิงประตูสูงสุด และอันดับสตรีคชนะติดต่อกันสูงสุด (คำนวณจากข้อมูลแมตช์ในระบบสายแข่งที่มีอยู่แล้ว ไม่ต้องกรอกข้อมูลเพิ่ม)
- **Hall of Fame แยกตามซีซั่น**: หน้า "หอเกียรติยศ" มีปุ่มสลับ "แชมป์ตลอดกาล" (แบบเดิม) กับ "แยกตามซีซั่น" ใหม่ — แสดงแชมป์/รองแชมป์ของทุกทัวร์นาเมนต์ จัดกลุ่มตามแท็กซีซั่นที่ตั้งไว้ตอนสร้างทัวร์นาเมนต์
- **กฎการแข่งขัน (Rule)**: แท็บใหม่ "กฎการแข่งขัน" แอดมินพิมพ์/แก้ไขเนื้อหากฎได้ตรงในเว็บเลย (ไม่ต้องแก้โค้ด) บันทึกลง Firestore แสดงผลทันทีให้ทุกคนเห็น
- **Match Verification Code**: ทุกแมตช์ในสายแข่งจะได้รหัสยืนยันคู่แข่งขันอัตโนมัติทันทีที่จับสาย (รูปแบบ `IPES-XXXX` สั้น ไม่ซ้ำกันทั้งระบบ) แสดงบนการ์ดแมตช์ทุกคู่ให้ทุกคนเห็น กดที่รหัสเพื่อคัดลอกได้ทันที ใช้ให้ผู้เล่นทั้งสองฝ่ายเช็กยืนยันว่าเจอคู่ที่ถูกต้องก่อนแข่ง

ระบบเดิมทั้งหมด (สร้างสาย/จับสาย, กรอกผล, Deadline, สถิติ, Achievement, Compare, Search, Backup & Restore, News) **ยังทำงานปกติทุกอย่าง ไม่กระทบกัน**

### รอบก่อนหน้า (สำหรับอ้างอิง)

ระบบจัดทัวร์นาเมนต์แบบสายแข่งจริง (Bracket / Single Elimination) ที่ทำงานครบวงจร:

- **สร้างสาย + จับสายแบบสุ่ม**: เลือกผู้เล่นจากฐานข้อมูล + ขนาดสาย (8/16/32/64/128) ระบบจับคู่สุ่มให้อัตโนมัติ ผู้เล่นไม่ครบจะได้ BYE ผ่านเข้ารอบถัดไปอัตโนมัติ
- **กรอกผล + เลื่อนสายอัตโนมัติ**: แอดมินกรอกสกอร์ (มีจุดโทษถ้าเสมอ) ผู้ชนะจะถูกเลื่อนเข้ารอบถัดไปให้อัตโนมัติ ไม่ต้องกดอะไรเพิ่ม จนถึงประกาศแชมป์/รองแชมป์ของทัวร์นาเมนต์เมื่อจบรอบชิง
- **Deadline ต่อรอบ + แก้ผลย้อนหลัง**: ตั้งเวลาปิดรับผลแต่ละรอบได้ ถ้าเลยเวลาแอดมินเลือก "ผู้ชนะ" ตรงได้โดยไม่ต้องมีสกอร์ และแก้ผลแมตช์ที่จบไปแล้วได้ — ระบบจะล้างผลทุกรอบถัดไปที่อิงจากผลเดิมให้อัตโนมัติ (รวมถึงถอนสถานะแชมป์ ถ้าแก้ไปถึงรอบชิง)
- **สถิติคำนวณสด**: ชนะ/แพ้/ประตู/Win Streak/Head-to-Head/ฟอร์มล่าสุด/**เทรนด์อัตราชนะ** (กราฟเส้นเล็ก 7/30/90 วัน/ทั้งหมด) คำนวณจากแมตช์จริงทุกครั้งที่โหลดหน้า ไม่มีการบันทึกตัวเลขซ้ำซ้อนไว้ที่ไหน
- **Achievement/Badge**: ติดป้ายอัตโนมัติ (แชมป์ครั้งแรก, ชนะติดต่อกัน 5/10, Giant Killer, แชมป์ติดต่อกัน ฯลฯ) ในหน้าโปรไฟล์ผู้เล่น
- **ไฮไลท์ Rivalry/Upset**: หน้าสถิติจะโชว์คู่ปรับที่เจอกันบ่อยและสูสี และแมตช์ที่ผู้เล่นอันดับต่ำกว่าพลิกชนะผู้เล่นอันดับสูงกว่า
- **เปรียบเทียบผู้เล่น (Compare)**: เทียบสถิติ 2 คนแบบเคียงข้าง + ประวัติการเจอกัน กดได้จากปุ่มในหน้าโปรไฟล์
- **ค้นหาแบบรวม (Global Search)**: ไอคอนแว่นขยายที่หัวเว็บ ค้นหาผู้เล่น/ทัวร์นาเมนต์/ข่าวพร้อมกันได้ในที่เดียว
- **Backup & Restore**: หน้า Admin มีปุ่ม Export ข้อมูลทั้งหมดเป็นไฟล์ JSON (และ Export CSV เฉพาะผู้เล่น) เก็บไว้ได้ทุกเมื่อ และนำไฟล์ JSON กลับมา Import (เขียนทับเฉพาะรายการที่ ID ตรงกัน ไม่ลบของเดิม) ใช้กรณีข้อมูลผิดพลาดและต้องการย้อนคืน
- **แดงแจ้งข่าวใหม่**: จุดสีแดงเล็ก ๆ ข้างแท็บ "ข่าว" เมื่อมีประกาศใหม่ที่ยังไม่ได้เปิดดู

ทัวร์นาเมนต์แบบเดิม (กรอกแชมป์/รองแชมป์ด้วยมือ สำหรับผลย้อนหลังก่อนมีระบบนี้) **ยังใช้งานได้ตามปกติ 100%** — ตอนกดปุ่ม "เพิ่มทัวร์นาเมนต์" จะมีให้เลือก 2 แบบคือ "สร้างสายแข่งใหม่" (ระบบใหม่) กับ "บันทึกผลย้อนหลัง" (แบบเดิม)

## 1. ⚠️ สำคัญ: build ทดสอบก่อน deploy

โค้ดรอบนี้แก้ไขผ่านเครื่องมือที่ไม่สามารถเชื่อมต่ออินเทอร์เน็ต/รัน `npm run build` ทดสอบให้ได้ จึงมีการตรวจสอบความถูกต้องของวงเล็บ/โครงสร้าง JSX, การเชื่อม import/export ระหว่างไฟล์, และชื่อ prop ทุกจุดที่แก้ไขด้วยมืออย่างละเอียดแล้ว แต่ **ขอแนะนำให้รัน `npm install && npm run build` บนเครื่องของคุณก่อน push ขึ้น Vercel ทุกครั้ง** เพื่อจับข้อผิดพลาดเล็ก ๆ ที่อาจหลุดรอดมาก่อนที่จะไป fail ตอน deploy จริง

```
npm install
npm run build
```

ถ้า build ผ่านบนเครื่อง ก็ push ได้อย่างมั่นใจ

## 2. ตั้งค่า Firebase

### 2.1 Deploy กฎ Firestore ใหม่ (ถ้ายังไม่ได้ deploy จากรอบก่อนหน้า)

ไฟล์ `firestore.rules` มี collection **`matches`** (เก็บผลแต่ละแมตช์ในสายแข่ง รวมถึงรหัสยืนยันคู่แข่งขัน `verifyCode` ที่เป็นแค่ field หนึ่งในเอกสารเดิม ไม่ต้องเพิ่มกฎอะไรใหม่) และ doc **`meta/rule`** (เนื้อหากฎการแข่งขัน) โดยใช้กฎเดียวกับของเดิม (อ่านได้ทุกคน, เขียนได้แค่ admin) ถ้ายังไม่เคย deploy กฎไฟล์นี้มาก่อนเลย ให้ทำตามนี้:

```
firebase deploy --only firestore:rules
```

(ต้องมี Firebase CLI ติดตั้งและ login ไว้แล้ว ถ้ายังไม่มีให้ `npm install -g firebase-tools` แล้ว `firebase login` และ `firebase use <project-id>` ก่อน)

หรือจะ copy เนื้อหาไฟล์ `firestore.rules` ไปแปะในหน้า Firestore > Rules บน Firebase Console แล้วกด Publish ก็ได้เช่นกัน ไม่ต้องใช้ CLI

### 2.2 ไม่ต้องสร้าง collection/document ล่วงหน้า

`matches`, `meta/rankingSnapshot`, และ `meta/rule` ทั้งหมดจะถูกสร้างขึ้นอัตโนมัติทันทีที่แอดมินใช้งานฟีเจอร์นั้นครั้งแรก (จับสายครั้งแรก / กดบันทึกอันดับครั้งแรก / บันทึกกฎการแข่งขันครั้งแรก) ไม่ต้องสร้างเองใน Console

### 2.3 ผู้เล่น/ทัวร์นาเมนต์เดิมที่มีอยู่แล้ว

ข้อมูลเดิมทั้งหมด (ผู้เล่น, ทัวร์นาเมนต์แบบกรอกมือ, ข่าว) ใช้งานได้ตามปกติ 100% ไม่ต้อง migrate อะไร ระบบจะรวมสถิติจากทัวร์นาเมนต์แบบสายแข่งใหม่เข้ากับตัวเลขเดิมให้อัตโนมัติทุกครั้งที่โหลดหน้า (คะแนน/แชมป์/รองแชมป์/Badge/อันดับยิงประตู/อันดับสตรีค จะถูกต้องโดยไม่ต้องแก้อะไรเพิ่ม) ส่วนรหัสยืนยันคู่แข่งขัน (`verifyCode`) จะมีเฉพาะแมตช์ที่จับสายตั้งแต่รอบอัปเดตนี้เป็นต้นไป — สายแข่งเก่าที่จับไว้ก่อนหน้านี้จะไม่มีรหัสนี้ (การ์ดแมตช์จะไม่โชว์ช่องรหัสถ้าไม่มี ไม่กระทบการใช้งานอื่น)

## 3. Push ขึ้น GitHub

```
git add .
git commit -m "Add recent matches, upcoming tournament, goals/streak ranking, season hall of fame, rule page, match verification code"
git push origin main
```

ถ้ายังไม่มี repo ให้สร้างใหม่บน GitHub แล้ว:

```
git init
git remote add origin <URL ของ repo>
git add .
git commit -m "Initial commit"
git branch -M main
git push -u origin main
```

## 4. Deploy ขึ้น Vercel

ถ้าโปรเจกต์เชื่อมกับ Vercel อยู่แล้ว (auto-deploy จาก GitHub) แค่ push ตามขั้นตอนที่ 3 ก็พอ Vercel จะ build และ deploy ให้อัตโนมัติ

ถ้ายังไม่ได้เชื่อม:

1. เข้า [vercel.com](https://vercel.com) → New Project → เลือก repo นี้จาก GitHub
2. ตั้งค่า Environment Variables ให้ตรงกับ `.env.local` ของเครื่อง (คีย์ทั้งหมดที่ขึ้นต้นด้วย `NEXT_PUBLIC_FIREBASE_`) — ต้องกรอกทีละตัวในหน้า Settings → Environment Variables ของโปรเจกต์ Vercel
3. กด Deploy

หลัง deploy ครั้งแรกแล้ว ทุกครั้งที่ push โค้ดใหม่ขึ้น branch `main`, Vercel จะ build และอัปเดตเว็บที่ live URL ให้อัตโนมัติ

## 5. สร้างบัญชี Admin

ใช้สิทธิ์ admin ตัวเดียวกับระบบเดิมทั้งหมด (รวมถึงสร้างสายแข่ง/กรอกผล/ตั้ง Deadline/Backup & Restore) ถ้ามี admin อยู่แล้วไม่ต้องทำซ้ำ ถ้ายังไม่มี:

1. Firebase Console → Authentication → Users → Add user (กรอกอีเมล/รหัสผ่าน)
2. Firestore Database → สร้าง collection ชื่อ `admins` → เพิ่ม document โดยใช้ **UID ของ user ที่สร้างในขั้นตอนที่ 1 เป็นชื่อ document** (ไม่ต้องมีข้อมูลภายในก็ได้ แค่มี document นั้นอยู่ก็พอ)

## 6. รันสคริปต์ตัวอย่างข้อมูล (ถ้าต้องการ)

```
node seed.mjs
```

สคริปต์นี้ seed เฉพาะ `players`, `tournaments`, `news` (แบบเดิม ไม่มีการ seed สายแข่ง `matches` เพราะการจับสายต้องเลือกผู้เล่นจริงจากฐานข้อมูลผ่านหน้าเว็บ) แต่ละ collection จะ seed แยกกันและข้ามอัตโนมัติถ้ามีข้อมูลอยู่แล้ว เหมาะสำหรับลองเว็บบนฐานข้อมูลที่ว่างเปล่า — หลัง seed ผู้เล่นแล้ว ลองกด "เพิ่มทัวร์นาเมนต์ → สร้างสายแข่งใหม่" จากหน้าเว็บเพื่อทดสอบระบบสายแข่งได้เลย

## 7. อัปเดตเนื้อหาเว็บ — **ไม่ต้อง push/deploy ใหม่**

ข้อมูลทั้งหมดที่เปลี่ยนบ่อย ๆ จัดการได้จากหน้าเว็บโดยตรง ผ่านปุ่ม login Admin มุมขวาบน **ไม่ต้องแก้โค้ดหรือ deploy ใหม่เลย**

อัปเดตผ่าน Admin UI ได้ทันที (real-time):
- เพิ่ม/แก้ไข/ลบผู้เล่น และรูปการ์ดผู้เล่น — ทำที่แท็บ "อันดับ"
- สร้างสายแข่งใหม่ + จับสาย (ได้รหัสยืนยันคู่แข่งขันให้อัตโนมัติทุกคู่), กรอกผล/แก้ผลย้อนหลัง, ตั้ง Deadline ต่อรอบ — ทำที่แท็บ "ทัวร์นาเมนต์" → เพิ่มทัวร์นาเมนต์ → สร้างสายแข่งใหม่
- เพิ่ม/แก้ไข/ลบทัวร์นาเมนต์แบบกรอกมือ (ผลย้อนหลัง) — ทำที่แท็บ "ทัวร์นาเมนต์" เช่นกัน
- ประกาศ/แก้ไข/ลบข่าว — ทำที่แท็บ "ข่าว" (ระบบจะประกาศข่าวอัตโนมัติให้เองเมื่อมีผลรอบรองชนะเลิศ/ชิงชนะเลิศของสายแข่ง)
- เขียน/แก้ไขกฎการแข่งขัน — ทำที่แท็บ "กฎการแข่งขัน" พิมพ์/แก้เนื้อหาได้ตรงในเว็บเลย ไม่ต้องแตะโค้ด
- บันทึก Ranking Snapshot (เพื่อให้ลูกศร ▲▼ ความเคลื่อนไหวอันดับทำงาน) และ Backup/Restore ข้อมูล (รวมกฎการแข่งขันด้วย) — ทำที่แท็บ "Admin"

แท็บ "สถิติยิง/สตรีค" และ "หอเกียรติยศ" (ทั้งแบบตลอดกาลและแยกซีซั่น) อัปเดตให้อัตโนมัติทุกครั้งที่มีผลแมตช์ใหม่หรือทัวร์นาเมนต์จบ ไม่ต้องกดอะไรเพิ่ม

ต้อง push code + deploy ใหม่ เฉพาะกรณี: แก้ดีไซน์/สี/เลย์เอาต์, เปลี่ยนสูตรคำนวณคะแนน, เพิ่มฟีเจอร์ใหม่ หรือแก้บั๊กในโค้ด — กรณีเหล่านี้เท่านั้นที่ต้องทำตามขั้นตอนที่ 3-4 ซ้ำ

## 8. (ทางเลือกในอนาคต) ฟีเจอร์ที่ตัดไว้ ถ้าอยากทำเพิ่มทีหลัง

ทุกอย่างในตาราง "สิ่งที่ตัด/ปรับให้เรียบง่ายลง" ที่ §0 สามารถทำเป็นเวอร์ชันเต็มได้ในอนาคตหากเว็บเติบโตจนคุ้มที่จะอัปเกรดเป็น Firebase Blaze plan (จ่ายตามใช้งาน) เช่น Cloud Functions สำหรับคำนวณ/แจ้งเตือนแบบ push จริง หรือเก็บ snapshot อันดับอัตโนมัติทุกวันเพื่อทำกราฟ Ranking History แบบเต็มรูปแบบ — ไม่กระทบกับการใช้งานปัจจุบันถ้ายังไม่ทำ

## 9. ทำทุกขั้นตอนนี้จาก iPad ได้ไหม (ไม่มีคอมพิวเตอร์/Terminal)

ได้ครับ ขั้นตอนที่ §1–§2 ปกติต้องใช้ Terminal (build ทดสอบ, git push, `firebase` CLI) ทำผ่าน Safari บน iPad ได้ทั้งหมด ด้วย 2 วิธีนี้:

### วิธีแนะนำ: GitHub Codespaces (เปิดผ่าน Safari ฟรี ไม่ต้องลงแอป)

ให้คอมพิวเตอร์เปล่า ๆ บนคลาวด์พร้อม VS Code + Terminal เต็มรูปแบบ เปิดในแท็บ Safari — ทำให้ build ทดสอบและ push ได้จริงเหมือนใช้คอมพิวเตอร์

1. แตะไฟล์ zip ที่ได้รับในแอป **Files** แล้วแตะค้างเลือก "แตกไฟล์ Zip" (Uncompress)
2. เข้า **github.com** ด้วย Safari → ล็อกอิน → เปิด repo เดิมที่ใช้กับเว็บนี้ (หรือสร้าง repo ใหม่ถ้ายังไม่มี)
3. ที่หน้า repo กดปุ่มสีเขียว **Code** → แท็บ **Codespaces** → **Create codespace on main** รอประมาณ 1-2 นาที จะได้หน้า VS Code เต็มรูปแบบพร้อม Terminal ด้านล่าง
4. เปิด Files app แบบ Split View ข้าง Safari แล้วลากไฟล์/โฟลเดอร์จาก `ipes` ที่แตกไว้ไปวางทับใน File Explorer ฝั่งซ้ายของ Codespace (ลากทับโฟลเดอร์ `app`, `components`, `lib` และไฟล์ที่แก้ เช่น `firestore.rules`, `DEPLOY_GUIDE.md`, `.gitignore` — **ข้ามไฟล์ `.env.local`**, ไม่ต้องอัปขึ้น GitHub เพราะมีคีย์ลับ — โฟลเดอร์นี้มี `.gitignore` ป้องกัน `.env.local` ไม่ให้ถูก commit โดยไม่ตั้งใจอยู่แล้ว แต่ปลอดภัยกว่าถ้าไม่ลากไฟล์นั้นไปเลยตั้งแต่ต้น)
5. พิมพ์ใน Terminal ของ Codespace:
   ```
   npm install
   npm run build
   ```
   ถ้า build ผ่าน (ไม่มี error สีแดง) แปลว่าโค้ดพร้อม deploy
6. commit + push จาก Terminal เดียวกัน:
   ```
   git add .
   git commit -m "Add recent matches, upcoming tournament, goals/streak ranking, season hall of fame, rule page, match verification code"
   git push
   ```
7. (ถ้าอยาก deploy firestore rules ผ่าน terminal นี้ด้วยเลย):
   ```
   npm install -g firebase-tools
   firebase login --no-localhost
   firebase deploy --only firestore:rules
   ```
   คำสั่ง `firebase login --no-localhost` จะให้ลิงก์มาเปิดในแท็บ Safari อีกแท็บเพื่อล็อกอิน แล้ว copy โค้ดยืนยันกลับมาวางที่ Terminal

### วิธีลัด: ไม่ต้องใช้ Codespaces เลย (เร็วกว่า แต่ไม่ได้ build ทดสอบก่อน)

- **Firestore rules**: เปิดไฟล์ `firestore.rules` ในแอป Files (แตะดูตัวอย่าง หรือเปิดด้วยแอปข้อความ) → คัดลอกทั้งหมด → เข้า Firebase Console บน Safari → Firestore Database → แท็บ **Rules** → ลบของเดิม วางอันใหม่ → **Publish** (ไม่ต้องใช้ CLI เลย วิธีนี้ใช้แทนขั้นตอนที่ §2.1 ได้)
- **GitHub โดยไม่ผ่าน Codespaces**: ที่หน้า repo บน Safari กด "Add file" → "Upload files" → ลากไฟล์/โฟลเดอร์จากแอป Files มาวาง → Commit (Safari บน iPad รองรับการลากทั้งโฟลเดอร์ได้ แต่ถ้าโครงสร้างไม่ตรง ให้อัปทีละโฟลเดอร์ย่อยแทน) — **ข้าม `.env.local` เช่นเดิม**
- **Vercel**: ถ้า repo เชื่อมกับ Vercel อยู่แล้ว แค่ push โค้ดขึ้น GitHub ด้วยวิธีใดวิธีหนึ่งข้างต้น Vercel จะ build และ deploy ให้อัตโนมัติ ดูผลได้ที่ vercel.com ในแท็บ Safari — ถ้า build fail จะเห็น log สีแดงบอกบรรทัดที่ผิด ส่ง log นั้นมาให้ผมดูต่อได้
- **Environment Variables**: ห้ามอัปไฟล์ `.env.local` ขึ้น GitHub เด็ดขาด ให้เปิดไฟล์นี้ดูค่าผ่านแอป Files แล้วพิมพ์ค่าทีละตัวเข้า Vercel → Settings → Environment Variables เอง (ทำครั้งเดียวตอนตั้งโปรเจกต์ ถ้าทำไปแล้วไม่ต้องทำซ้ำ)

แนะนำให้ใช้ Codespaces (วิธีแรก) ถ้าเป็นไปได้ เพราะได้ build ทดสอบจริงก่อน push ลดโอกาส deploy แล้วพังที่ Vercel
